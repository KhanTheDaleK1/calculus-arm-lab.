# Contributing to CalculusArm Dynamics Lab

We welcome contributions to the CalculusArm Dynamics Lab project! To ensure a smooth and effective collaboration, please follow these guidelines.

## How Can I Contribute?

### Reporting Bugs

*   Ensure the bug hasn't already been reported.
*   Clearly describe the bug, including steps to reproduce it, expected behavior, and actual behavior.
*   Provide any relevant error messages or logs.

### Suggesting Enhancements

*   Explain why this enhancement would be useful.
*   Describe the proposed solution in detail.

### Code Contributions (Web)

*   For web-related changes (HTML, CSS, JavaScript), please ensure your code adheres to standard web development practices.
*   Use `npm` for dependency management in the `web` directory.
*   Test your changes thoroughly in multiple browsers if applicable.

### Code Contributions (Python)

*   For Python scripts, ensure you have a `requirements.txt` file listing all dependencies.
*   Adhere to PEP 8 style guidelines.
*   Include unit tests for new functionality.

### Code Contributions (Arduino Firmware)

*   For Arduino firmware, ensure your code compiles without errors and is well-commented.
*   Provide clear instructions on how to upload the firmware.
*   Test your firmware thoroughly on the target hardware.

## Code of Conduct

Please note that this project is released with a Contributor Code of Conduct. By participating in this project, you agree to abide by its terms.

Thank you for contributing!
